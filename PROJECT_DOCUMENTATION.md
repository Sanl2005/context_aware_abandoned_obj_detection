# Project Documentation: Context-Aware Abandoned Object Detection

This document provides a detailed explanation of the project structure, the role of each file, and how the overall system works.

## System Overview

The system is a distributed application designed to detect abandoned objects in video feeds (CCTV) and send real-time alerts. It consists of three main components:

1.  **ML Service (Python/Flask)**: Handles real-time object detection using YOLOv8, tracks object abandonment, and captures images of alerts.
2.  **Rails Backend (Ruby on Rails)**: Serves as the central API, manages persistent storage in PostgreSQL, and handles real-time data broadcasting via ActionCable.
3.  **Frontend Dashboard (React)**: Provides a cyber-security surveillance styled UI for viewing live feeds, archived alerts, and managing objects.

## Technical Implementation Details

### Detection & Abandonment Logic
The ML service uses YOLOv8 to identify objects. An object is considered "abandoned" if it remains stationary (tracked by ID) while no person is detected within its proximity for a threshold duration (defined in `risk_profile.json`).

### Image Capture
When an object is confirmed as abandoned:
1. The ML service captures a crop of the object and the last person who interacted with it.
2. These are sent as Base64 strings to the Rails API.
3. Rails saves the record and uses **ActiveStorage** to store the images on disk.

### SOS Alerts
If an object's confidence exceeds 80% for more than 10 seconds, `alert_service.py` triggers:
- **Email**: via Gmail SMTP.
- **SMS**: via Fast2SMS API.

### Hardware Requirements
- NVIDIA GPU recommended for optimal frame rates (using CUDA).
- Minimum 8GB RAM.
- Web-camera or RTSP stream support.
